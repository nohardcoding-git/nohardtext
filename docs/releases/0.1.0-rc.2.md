# NoHardText 0.1.0-rc.2 Release Notes

## Summary

NoHardText `0.1.0-rc.2` is the first release candidate for the NoHardText CLI and core detection packages.

NoHardText detects hardcoded user-facing UI strings in React / TSX code and helps teams answer:

> Can I ship this release without hardcoded text?

## Highlights

- React / TSX hardcoded text detection.
- Built-in localization and accessibility rules.
- CLI scan command.
- Human-readable terminal report.
- JSON report output.
- GitHub Actions annotation output.
- CI failure threshold support.
- Config file validation.
- Rule enable/disable config.
- Rule severity overrides.
- Custom component text prop detection.
- Localization health score.
- Ship decision summary.
- Rule breakdowns.
- Category breakdowns.
- Top issues summary.
- Release safety checks.
- Package pack checks.

## Current Rules

| Rule ID | Rule | Category | Default Severity |
|---|---|---|---|
| NHT1001 | JSX Text | localization | high |
| NHT1002 | Placeholder Attribute | localization | high |
| NHT1003 | Title Attribute | localization | high |
| NHT1004 | ARIA Label | accessibility | high |
| NHT1005 | Alt Attribute | accessibility | high |
| NHT1006 | Component Text Prop | localization | high |

## CLI Usage

```bash
nohardtext scan src
nohardtext scan src --json
nohardtext scan src --json --output nohardtext-report.json
nohardtext scan src --github-annotations --fail-on high
nohardtext rules
nohardtext --help
nohardtext --version
```

## Config Example

```json
{
  "ignore": [
    "storybook-static"
  ],
  "failOn": "high",
  "componentTextProps": [
    "message",
    "text"
  ],
  "rules": {
    "NHT1001": "high",
    "NHT1002": "medium",
    "NHT1003": "off"
  }
}
```

## Known Limitations

- React / TSX is the primary supported target.
- Auto-fix is not implemented yet.
- VS Code extension is not implemented yet.
- MCP server is not implemented yet.
- Hosted reporting is not implemented yet.
- Framework-specific integrations beyond the CLI are not included yet.
